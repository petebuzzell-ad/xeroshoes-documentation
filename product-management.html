Media Requirements

Image standards ensuring brand consistency across all product seasons and categories.

Technical Specifications

  • Dimensions: 1540x1000 pixels (rectangular format, 1.54:1 aspect ratio)
  • Resolution: 86 DPI (web-optimized)
  • Format: JPEG, progressive encoding
  • File naming: Follows XeroShoes internal conventions (some images include "_WEB" suffix indicating web-optimized versions)

Design Consistency Standards

Brand Consistency: These standards ensure visual consistency across product seasons and maintain professional brand presentation.
  • Background: Consistent neutral background (typically white or light gray)
  • Lighting: Even, professional studio lighting with minimal shadows
  • Product positioning: Centered composition with consistent whitespace margins
  • Color accuracy: True-to-life color representation for accurate customer expectations
  • Sharpness: High-resolution, crisp detail shots for texture and material visibility

File Organization

Standardized naming convention for efficient asset management:

  • Format: [PRODUCT-CODE]_[COLOR]_[ANGLE]_[SHOT-NUMBER]_WEB_[UUID].jpg
  • Example: CAPY-NGBU_Dillon-Canvas-Slip-On-Youth_Insignia-Blue_Front_7406_WEB_b7ade370-cf5e-427d-950d-d74f83eeafc3.jpg
  • Note: "_WEB" suffix appears in some filenames as part of XeroShoes' internal asset management system

Quality Assurance

  • Consistency check: All angles must maintain same lighting and background
  • Color matching: Verify color accuracy across all product angles
  • Completeness: Ensure all required angles are captured for each product
  • File optimization: Verify web optimization and loading performance

Field Ownership & Management

Understanding which fields are user-editable versus app-controlled is critical for effective product management.

User-Editable Fields

Management: These fields should be updated by content managers and product teams through Shopify Admin or bulk import tools.
  • Core Product Data: All custom.* metafields (display_title, feed_description, product_highlights, etc.)
  • Content Metaobjects: Product descriptions, media blocks, FAQs, care instructions
  • SEO Fields: title_tag, description_tag
  • Variant Control: custom.draft_variant (critical for visibility management)
  • Google Shopping: mm-google-shopping.* fields (for product feed optimization)

App-Controlled Fields

Read-Only: These fields are automatically managed by apps and should not be manually edited.
  • Reviews & Ratings: reviews.*, stamped.* (managed by review apps)
  • Discovery: shopify--discovery--* (managed by Shopify Search & Discovery)
  • Tax Integration: avalara.taxcode (managed by Avalara app)
  • Bundles: simple_bundles.* (managed by Simple Bundles app)
  • Facebook Marketing: mc-facebook.* (managed by Facebook app)

Shopify Taxonomy Fields

Standardized: These use Shopify's built-in taxonomy system and should be managed through the standard product interface.
  • Color/Pattern: shopify.color-pattern (use Shopify's color picker)
  • Size: shopify.shoe-size (use standardized size values)
  • Gender: shopify.target-gender (use predefined options)
  • Age Group: shopify.age-group (use standard age ranges)
  • Material: shopify.clothing-accessory-material (use material taxonomy)

Critical Metafield: custom.draft_variant

The custom.draft_variant metafield is a custom implementation that extends Shopify's native variant management capabilities. This metafield provides granular control over variant visibility on product detail pages.

Important: This is a custom metafield that directly impacts customer experience and product management workflows. Understanding its behavior is critical for effective product management.

Variant Visibility Control

  • TRUE: Variant is excluded from the product detail page
  • FALSE or blank: Variant is included and visible on the product detail page

Use Cases

  • Seasonal Products: Hide out-of-season variants without deleting them
  • Inventory Management: Hide variants that are temporarily out of stock
  • Product Testing: Test new variants without full public visibility
  • Regional Availability: Hide variants not available in specific regions
  • Product Lifecycle: Manage product discontinuation process

Implementation Notes

  • Metafield applies to individual variants, not entire products
  • Changes take effect immediately upon saving
  • Hidden variants are not accessible via direct URL
  • Variant data remains intact when hidden (pricing, inventory, etc.)
  • Admin users can still see and manage hidden variants in Shopify Admin

Technical Implementation

The custom.draft_variant metafield is implemented through a sophisticated Liquid + JavaScript system that dynamically controls variant visibility on product pages.

Technical Details: For complete code examples, file locations, and implementation details, see Theme Architecture → Custom Draft Variant Implementation.
Implementation Overview
  • Liquid Processing: Templates process variant data and generate JavaScript configuration
  • JavaScript Engine: Client-side processing handles dynamic visibility control
  • Multi-Option Logic: Complex cascading logic for multiple variant options
  • Performance Optimized: Efficient filtering and state management

Variant Visibility Management

Managing variant visibility using the custom.draft_variant metafield - a custom implementation specific to XeroShoes that extends beyond standard Shopify functionality.

Variant Visibility Control

  1. Access product in Shopify Admin
  2. Navigate to the Variants section
  3. Select the variant to modify
  4. Scroll to the Metafields section
  5. Locate the custom.draft_variant metafield
  6. Set value:
    • TRUE: To hide variant from product page
    • FALSE or leave blank: To show variant on product page
  7. Save the product
  8. Verify visibility changes on the live product page
  9. Apply same changes to corresponding instance if needed
Pro Tip: Use variant visibility management for seasonal inventory control, testing new variants, or managing regional availability without deleting variant data.

Additional Use Cases

  • Seasonal Product Management: Hide winter boots during summer without losing variant data
  • A/B Testing: Test new color variants with limited visibility before full launch
  • Regional Rollouts: Gradually introduce products to different markets
  • Inventory Transitions: Manage product discontinuation while maintaining data integrity
