Content Data Standards

Minimum content requirements for publication across all business functions. Every piece of content must meet these standards before going live.

Minimum Required Data

Content cannot be published without completing all items in this checklist:

Core Page Data

Field Required For Minimum Standard Validation
Page Title SEO, Navigation, Legal Clear, descriptive, brand-consistent 50-60 characters, includes "XeroShoes" when appropriate
URL Handle SEO, Technical SEO-friendly, lowercase, hyphenated No spaces, special characters, or uppercase letters
Page Content Marketing, Legal, SEO Complete, accurate, benefit-focused Must include key information and call-to-action
SEO Title SEO, Marketing Optimized for search engines 50-60 characters, includes target keywords
Meta Description SEO, Marketing Compelling description for search results 150-160 characters, includes call-to-action

Blog Post Data

Field Required For Minimum Standard Validation
Blog Title SEO, Marketing, Brand Engaging, keyword-optimized 50-60 characters, includes target keywords
Author Legal, Brand Valid author assignment Must be existing team member or approved contributor

Blog Content Marketing, SEO, Legal Complete, valuable, well-structured Minimum 300 words, includes headings and formatting

Content Metafields & Custom Data

XeroShoes uses metafields to store structured content data, SEO information, and custom attributes for pages and blog posts.

Field Ownership & Management

Understanding who controls and manages different content fields:

Field Ownership:
  • User-Editable: Content team can modify directly in Shopify Admin
  • App-Controlled: Managed by third-party apps (SEO, analytics, etc.)
  • System-Generated: Automatically created by Shopify or theme

SEO Metafields

Specialized metafields for search engine optimization:

SEO Metaobject Integration: The theme uses category_meta_data_meta_title_and_description metaobjects to automatically apply SEO data based on blog post tags. This ensures consistent SEO optimization across content categories.

Automatic SEO Application

Trigger Action Result
Blog post with tag Theme checks metaobject Applies custom title and description
Tag matches category Uses metaobject data Consistent SEO across category
No tag match Uses default values Fallback to shop name/description

Content Block Configuration

How Praella has configured XeroShoes content blocks for optimal brand consistency and user experience:

Barefoot Image with Text Configuration

Purpose: Educational content about barefoot shoes with dual-image layout

Praella Configuration Example:

Heading Configuration:
  • Text: "Barefoot Shoes" (H2 heading)
  • Font Family: font-family-heading (Paralucent)
  • Alignment: Left-aligned on desktop and mobile
  • Spacing: No top margin, 2-unit bottom margin
Content Configuration:
  • Text: "Barefoot Shoes give you that natural, free, fun feeling of being in bare feet — letting your toes spread, and your feet bend and move and flex — while letting you safely FEEL the world beneath you. All those sensors in your soles give your brain the feedback it needs to help your body move better, efficiently, and enjoyably. Slip on a pair of Xero Shoes and Live Life Feet First!"
  • Font Family: font-family-base (Inter)
  • Text Size: fs-base (standard body text)
  • Font Weight: fw-normal (regular weight)
  • Alignment: Left-aligned

Global Page Header Banner Configuration

Purpose: Hero banners for category pages with responsive images

Praella Configuration Example (Barefoot Shoes Page):

Image Configuration:
  • Desktop Image: "Best-Minimalist-barefoot-shoes.jpg"
  • Mobile Image: "Best-Minimalist-barefoot-shoes-m.jpg"
  • Container Width: fluid_layout (full width)
  • Overlay: Disabled for clean image display
Layout Configuration:
  • Display: d-flex (flexbox layout)
  • Direction: flex-md-row (horizontal on desktop)
  • Mobile Direction: flex-column (vertical on mobile)
  • Alignment: Center-aligned content
  • Height: medium (standard banner height)

Custom Section Configuration

Purpose: Flexible content containers with precise spacing and layout control

Praella Configuration Example:

Container Configuration:
  • Container Width: container (standard width)
  • Display: d-flex (flexbox layout)
  • Direction: flex-md-column (vertical on desktop)
  • Alignment: align-items-md-start (left-aligned)
  • Justify Content: justify-content-md-start (top-aligned)
Spacing Configuration:
  • Desktop Padding: pt-md-9, pb-md-9 (large top/bottom padding)
  • Mobile Padding: pt-5, pb-1 (medium top, small bottom)
  • Horizontal Padding: ps-md-3, pe-md-3 (small left/right padding)
  • Gap Control: No gaps between elements

Configuration Best Practices

Element XeroShoes Standard Purpose

Body Text font-family-base, fs-base, fw-normal Readability with Inter font
Spacing Consistent margin/padding classes Visual rhythm and hierarchy
Images Desktop + mobile variants Responsive optimization
Containers container, small_container_layout Content width consistency
Configuration Guidelines: When configuring content blocks, follow XeroShoes established patterns for spacing, typography, and responsive behavior to maintain brand consistency across all pages.

Responsive Image Breakpoints

The theme automatically generates responsive images at these breakpoints:

Responsive Image Sizes:
  • 350w: Mobile small screens
  • 550w: Mobile large screens
  • 750w: Tablet portrait
  • 991w: Tablet landscape
  • 1200w: Desktop small
  • 1500w: Desktop large

Image Optimization Settings

Setting Value Purpose
Loading Strategy Lazy loading (default) Performance optimization
Preload Priority High for above-fold images Critical rendering path
Alt Text Required, descriptive Accessibility compliance
File Size < 500KB per image Loading performance
Format Priority WebP, then JPG Modern browser optimization

Mobile Image Requirements

For content blocks that support separate mobile images:

Mobile Image Specifications:
  • Naming Convention: Add "-m" suffix (e.g., "hero-image.jpg" → "hero-image-m.jpg")
  • Dimensions: 750px width minimum, 1200px recommended
  • Aspect Ratio: 16:9 or 4:3 for mobile optimization
  • File Size: < 300KB for mobile performance
  • Content: Cropped/optimized for mobile viewing

Video Specifications

Video Type Format Resolution Duration File Size
Background Video MP4 1920x1080 10-30 seconds < 5MB
Product Video MP4 1280x720 30-60 seconds < 10MB
UGC Video MP4 1080x1080 15-45 seconds < 8MB

Image Naming Conventions

XeroShoes Naming Standards:
  • Category Pages: "Best-[Category]-[Type].jpg" (e.g., "Best-Minimalist-barefoot-shoes.jpg")
  • Mobile Variants: Add "-m" suffix (e.g., "Best-Minimalist-barefoot-shoes-m.jpg")
  • Product Images: "[STYLE]-[COLOR]_[Product-Name]_[Color-Name]_[Size].jpg"
  • Blog Images: "[Topic]-[descriptive-name].jpg"
  • Banner Images: "[Page-Name]-[descriptive].jpg"

Accessibility Requirements

Requirement Specification Implementation
Alt Text Descriptive, contextual Required for all images
Color Contrast WCAG 2.1 AA compliant 4.5:1 minimum ratio
Text Overlay Readable on all backgrounds Use overlay or text shadows
Focus Indicators Visible keyboard navigation Theme default styling
Image Quality Guidelines: All images should be high-quality, professionally shot, and optimized for web delivery. Use consistent lighting, composition, and styling to maintain brand visual consistency across all content blocks.
