Overview

This document provides a comprehensive mapping between metafields used on the US XeroShoes site (xeroshoes.com) and the EU XeroShoes site (shopxeroshoes-eu.myshopify.com). The analysis reveals significant differences in metafield availability and namespace conventions:

Key Findings:
  • US Site: 61 metafields (14,054 products¹)
  • EU Site: 33 metafields (9,518 products¹)
  • Shared Fields: 8 metafields (identical namespaces)
  • US-Only Fields: 53 metafields (not available on EU site)
  • Namespace-Mapped Fields: 25 fields with different namespaces (custom.* vs product.*)
Data Export Dates:
  • US Data: October 20, 2025 (14,054 products¹)
  • EU Data: October 23, 2025 (9,518 products¹)

Data Mapping Process

Step-by-step process for mapping data between US and EU sites:

1. Namespace Translation

Translation Rules:
  • US → EU: Replace custom. with product.
  • EU → US: Replace product. with custom.
  • Exceptions: custom.comfort_section remains the same on both sites
  • Variant Exception: custom.draft_variant → product_variant.draft_variant
  • App Metafields: No translation needed (identical namespaces)

2. Data Validation

Step Description Validation
1 Extract metafields from source site Verify field names and data types
2 Apply namespace translation Check namespace mapping accuracy
3 Validate data types match Ensure data type compatibility
4 Check for missing fields Identify fields not available on target site
5 Test data import Verify successful import and display

3. Mapping Rules Summary

Key Mapping Rules:
  • US → EU: Replace custom. with product.
  • EU → US: Replace product. with custom.
  • Exception: custom.comfort_section remains the same on both sites
  • Variant Exception: custom.draft_variant → product_variant.draft_variant
  • App Metafields: No translation needed (identical namespaces)
  • US-Only Fields: Cannot be mapped to EU site (53 fields unavailable)

Implementation Notes

Data Synchronization Challenges

Critical Issues:
  • Field Availability: 53 US-only fields cannot be mapped to EU site
  • Field Count Mismatch: US has 61 metafields vs EU has 33 metafields
  • Data Completeness: EU site missing significant functionality (reviews, Google Shopping, tax compliance)
  • Metaobject References: Ensure referenced metaobjects exist on target site
  • File References: Verify image/file assets are available on target site

Testing Requirements

Test Type Description Validation
Namespace Mapping Verify correct namespace translation All mappable metafields use correct namespace
Data Type Validation Ensure data types match between sites No data type conversion errors
Content Display Verify content displays correctly on frontend All metafield content renders properly
Metaobject References Test metaobject reference integrity All metaobject references resolve correctly
Missing Field Handling Test behavior when fields don't exist on target Graceful handling of unavailable fields

Error Handling

Common Issues and Solutions:
  • Missing Metaobjects: Create missing metaobjects before importing data
  • File References: Upload missing image files to target site
  • Data Type Mismatches: Convert data to match target site data types
  • Namespace Conflicts: Use mapping table to resolve conflicts
  • Unavailable Fields: Skip or provide alternative data for US-only fields

Recommendations

Best Practices:
  • Incremental Sync: Start with shared fields, then add site-specific fields
  • Data Validation: Implement comprehensive validation before data transfer
  • Rollback Plan: Maintain ability to revert changes if issues occur
  • Monitoring: Track data integrity and display issues post-sync
  • Documentation: Maintain this mapping document as the source of truth
